Loterio

A decentralized lottery system on blockchain.


Description

Loterio is a lottery system using the blockchain technology working within the Ethereum Virtual Machine.

As any other lottery, it has a start, pot, participants, an end, and a winner, but some mechanisms have been implemented for decentralization and governance.

In summary, the lottery process works as follows:

A lottery process is started, then users can start to participate. Pot is increased with each participation. After a specified period, the lottery ends, entries are closed, and a winner is chosen. The winner receives the accumulated pot, and a new lottery can be started.


Participation

Connect to the lottery using a web3 wallet. (Metamask, for example)

Choose one of the networks available. Each lottery at each network work independently from each other.

Now you can participate calling the participate() function in the contract or directly from the webpage.

* The status of the lottery has to be active to participate.


Rules and concepts

Pot is increased with each participation.

Blockchain cannot manage automatic processes, someone has always to call a function from contract in order to perform an action. That means, that someone has to start and end the lottery process each time.

To ensure that, two additional incentives have been implemented: 

  • Start incentive: A reward given to the user that starts the lottery.

  • End incentive: A reward given to the user that ends the lottery.

With them, when a user performs any of these actions, they receive a reward for it, doing profitable for them ending a current lottery or starting a new one, and ensuring the lottery have an end and a new one is always started.

Alright, set a rule for starting a new lottery is simple, when one is over, you can start the next one. But, what about to be able to end it?

In order to prevent users to end it up earlier than the end date, each time a user tries to do it, the blockchain looks for the start date and the period stablished for the current lottery, and then it decides if it’s time or not to finish it. Users can look for it using the checkEnd() function in the contract, that let you know when a lottery can be finished. If a user tries to do it earlier than the expected end date, the action will be rejected by the contract and not performed.

In addition, in each lottery, a pot for the next lottery is also accumulated. That way, there is always a good reason to participate, even you are the only one in it!

All these incentives are directly dependent of the participations. The cost of each entry is not used only to increase the current pot, but to ensure these incentives.

We can find the next ratios, equivalent to the proportion provided to each incentive/reward:

  • Pot ratio: Percentage of the entry used to increase the pot.

  • Start ratio: Percentage of the entry allocated to the start incentive.

  • End ratio: Percentage of the entry allocated to the end incentive.

  • Init ratio: Percentage of the entry used to accumulate the pot for next lottery.

Despite these, any pot and incentive can be increased, if necessary, to avoid unwanted states. In addition, lottery can be set under maintenance, preventing a new lottery to start and allowing ratios to be modified.

As for the rewards obtained by users (winning, starting or ending a lottery), they just accumulate and can be claimed at any time using the claim() function in the contract or directly from the webpage.
